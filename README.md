# browser

[![Go Version](https://img.shields.io/badge/Go-1.26+-00ADD8?style=flat&logo=go)](https://golang.org)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![GoDoc](https://pkg.go.dev/badge/github.com/unitsvc/browser.svg)](https://pkg.go.dev/github.com/unitsvc/browser)

Helpers to open URLs, readers, or files in the system default web browser.

## Features

- Cross-platform support (Linux, macOS, Windows, *BSD)
- Simple API for opening URLs, files, and readers
- Proper error handling and resource cleanup
- Zero dependencies (except `golang.org/x/sys` for Windows support)

## Installation

```bash
go get github.com/unitsvc/browser
```

## Usage

### Open a URL

```go
import "github.com/unitsvc/browser"

err := browser.OpenURL("https://go.dev")
if err != nil {
    log.Fatal(err)
}
```

### Open a local file

```go
err := browser.OpenFile("index.html")
if err != nil {
    log.Fatal(err)
}
```

### Open from a reader

```go
import "strings"

html := strings.NewReader("<html><body>Hello!</body></html>")
err := browser.OpenReader(html)
if err != nil {
    log.Fatal(err)
}
```

## Platform Support

| Platform | Browser Detection |
|----------|-------------------|
| Linux | `xdg-open`, `x-www-browser`, `www-browser`, `wslview` |
| macOS | `open` command |
| Windows | ShellExecute API |
| FreeBSD | `xdg-open` |
| OpenBSD | `xdg-open` |
| NetBSD | `xdg-open` |

## Use Cases

- **Documentation**: Open local HTML documentation in a browser
- **Dashboards**: Launch web-based dashboards from CLI tools
- **Reports**: Display HTML reports generated by your application
- **OAuth Flows**: Initiate browser-based authentication
- **Testing**: Preview generated HTML content during development

## Example

See the [examples](examples/) directory for a complete command-line tool:

```bash
# Open a URL
go run examples/Open/main.go https://go.dev

# Open a file
go run examples/Open/main.go index.html

# Open from stdin
echo "<html><body>Hello</body></html>" | go run examples/Open/main.go -
```

## Notes

- The choice of browser is entirely client-dependent
- Temporary files created by `OpenReader` are automatically cleaned up on error
- On Linux, multiple browser providers are tried in order
- Headless environments may not have a default browser configured
